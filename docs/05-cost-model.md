## 1. 概述

**目的**：成本透明化，非预算限制。所有价格基于阿里云公开价格估算，并同步给出打一折后的内部价格。实际单价栏位留空供替换。

**数据规模假设**：
- 总量 100PB，增量每天 TB 级
- 原始数据保留 10 年
- GPU 实例以 A100-80GB 为基准

**重要声明**：以下为方案阶段估算，误差 ±30%。实际成本需在 Phase 1 小规模 benchmark 后修正。

## 2. 存储成本

### 2.1 S3 兼容存储

| 存储层 | 数据量（稳态） | 存储类别 | 公开单价（元/GB/月） | 打一折（元/GB/月） | 月成本（公开） | 月成本（一折） |
|---|---|---|---|---|---|---|
| **raw**（原始数据） | 50PB | 标准存储（30天）→ 低频存储 → 归档存储 | 0.12 → 0.08 → 0.015 | 0.012 → 0.008 → 0.0015 | ¥500万 → ¥50万 | ¥50万 → ¥5万 |
| **bronze**（粗过滤后） | 30PB | 低频存储（90天后） | 0.08 | 0.008 | ¥240万 | ¥24万 |
| **silver**（加工后，近30天） | 5PB | 标准存储 | 0.12 | 0.012 | ¥60万 | ¥6万 |
| **silver**（加工后，历史） | 15PB | 低频存储 | 0.08 | 0.008 | ¥120万 | ¥12万 |
| **silver_emb**（Lance 向量+索引） | 10PB | 标准存储 | 0.12 | 0.012 | ¥120万 | ¥12万 |
| **gold**（训练数据集） | 3PB | 标准存储 | 0.12 | 0.012 | ¥36万 | ¥3.6万 |

> **说明**：raw 数据由标准存储 → 低频 → 归档自动降级。以上拆分已计入降级时间线和各阶段单价。

### 7.2 成本占比分析（公开价格）


## 8. 成本优化建议

| 优化项 | 预期节省 | 实施难度 |
|---|---|---|
| **raw 数据加速降级**：30天 Standard → 立即转低频 → 90天转归档 | 存储成本降低 30-40% | 低（修改 S3 生命周期规则） |
| **Spot 实例**：Spark CPU 节点 80% 使用 Spot 实例 | Spark 计算成本降低 60% | 低（K8s 配置 spot 优先） |
| **Embedding 模型剪枝/量化**：Phase 1 使用 ONNX 量化版本 | Embedding GPU·h 降低 30% | 中（需验证精度损失） |
| **压缩率优化**：Parquet ZSTD level 6（比 level 3 多节省 10% 空间） | 存储成本降低 5-10% | 低（Spark 配置项） |
| **去重先行**：在 Embedding 之前先做 MinHash 去重，减少重复数据的 embedding 计算 | Embedding GPU·h 降低 15-25% | 中（调整 pipeline 顺序） |
| **冷数据 Compaction**：合并历史小文件为 512MB+ 大文件，减少 LIST 请求量 | 请求费用降低 50% | 低（已有 Compaction 调度） |
